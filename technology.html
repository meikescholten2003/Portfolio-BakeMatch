Technology Stack

An overview of the technologies used to build BakeMatch and the reasoning behind each choice.

Core Technologies

BakeMatch uses core web technologies chosen for their learning value, creative control, and project realism. The technology stack focuses on fundamental skills while creating a functional, user-friendly platform.

HTML5

Purpose: Structure and semantic content organization for all pages.

Why chosen: HTML5 provides the foundation for web content. Learning to structure pages properly with semantic elements like header, nav, section, and article ensures accessibility and clear organization. This project required building multiple interconnected pages from scratch.

Key usage: Created the homepage with ingredient selection, recipe overview pages, recipe detail pages, favorites section, tips pages, and blog pages. Semantic markup ensures screen readers can navigate content effectively.

CSS3

Purpose: Visual design, layout, branding, and responsive behavior.

Why chosen: CSS was chosen to implement the complete design system defined in the style guide. Modern CSS features like Flexbox and Grid enable complex layouts without additional libraries. Custom properties allow consistent theming throughout the platform.

Key usage: Implements the warm beige and brown color scheme, rounded UI components, typography hierarchy, spacing system, and grid layouts. Ensures visual consistency across all pages and maintains the cozy, approachable atmosphere central to BakeMatch's identity.

JavaScript (ES6+)

Purpose: Interactive functionality, dynamic content updates, and data management.

Why chosen: JavaScript makes BakeMatch interactive. Core features like ingredient filtering, recipe loading, favorites management, and navigation depend on JavaScript.

Key usage: Handles ingredient selection and filtering, fetches recipe data from the external API, manages favorites using LocalStorage, enables search functionality, and controls the AI chatbot interface. Uses async/await for API calls.

Figma

Purpose: Design, prototyping, and style guide creation.

Why chosen: Figma was the main design tool for creating wireframes, low-fidelity and high-fidelity prototypes, and the complete style guide. Previous experience with Figma allowed focus on UX/UI principles rather than learning a new tool.

Key usage: Created all visual designs, tested interaction flows, defined typography and color systems, and produced prototypes for user testing. Figma's component system helped maintain design consistency.

Visual Studio Code

Purpose: Code editor and development environment.

Why chosen: VS Code was the main code editor for writing and organizing HTML, CSS, and JavaScript files. Features like syntax highlighting, file management, and live preview workflow improved efficiency and code readability.

Key usage: Organized project file structure, wrote and edited all code files, debugged JavaScript, and tested the website during development. Extensions helped with code formatting and error detection.

APIs and External Services

TheMealDB API

Purpose: External recipe database providing real recipe data.

Why chosen: Instead of creating a custom database or using static JSON files, TheMealDB provides professional recipe data including categories, ingredients, instructions, and images. This makes the project more realistic and scalable while teaching API integration.

Implementation: Fetch requests retrieve recipe data dynamically. The API provides endpoints for searching by ingredient, browsing categories, and getting detailed recipe information. This reflects how real recipe platforms work in practice.

Local Storage API

Purpose: Client-side data persistence for favorite recipes.

Why chosen: LocalStorage allows users to save favorite recipes directly in the browser without creating accounts or backend infrastructure. This fits the project scope while still providing a useful, practical feature.

Implementation: Stores and retrieves favorite recipe IDs as JSON data. Users can add or remove favorites at any time, and the data persists across browser sessions. This demonstrates understanding of browser storage capabilities.

Fetch API

Purpose: HTTP requests for retrieving recipe data from TheMealDB.

Why chosen: The Fetch API is the modern standard for making HTTP requests in JavaScript. It returns promises that work naturally with async/await syntax, making asynchronous code more readable.

Implementation: All communication with TheMealDB uses fetch requests. Error handling catches failed requests and provides feedback when recipes cannot be loaded. This ensures the interface remains responsive even when network issues occur.

AI Chatbot Integration

Purpose: Conversational assistance for cooking questions and ingredient tips.

Why chosen: An AI chatbot adds practical value by helping users with ingredient substitutions, cooking techniques, and recipe questions. The focus was on UI integration and user experience rather than building the AI model.

Implementation: JavaScript-based chat interface connects to an external AI API. The chatbot appears as a floating panel accessible from any page, providing help without interrupting the recipe browsing experience.

Development and Design Tools

Figma

Purpose: UI/UX design and prototyping.

Used for creating wireframes, low-fidelity and high-fidelity prototypes, moodboards, and the complete style guide. Figma's collaborative features and component system helped maintain design consistency throughout the project.

VS Code

Purpose: Code editor and development environment.

Primary tool for writing HTML, CSS, and JavaScript. Offered excellent support for web development with syntax highlighting, integrated debugging, file management, and extensions for code formatting.

Browser DevTools

Purpose: Debugging and testing.

Essential for inspecting DOM elements, debugging JavaScript, testing responsive design, analyzing network requests, and monitoring console errors. Used extensively during development and testing phases.

Trello

Purpose: Project planning and task management.

Organized the development process into clear steps and tracked progress throughout the project. Helped maintain focus and ensure all planned features were completed systematically.

Pinterest

Purpose: Visual inspiration and research.

Used for gathering design inspiration, exploring UI patterns, and collecting references for the moodboard. Helped define the visual direction and discover effective design patterns for recipe platforms.

Google Gemini

Purpose: Persona image generation.

Generated realistic profile photos for user personas using text prompts describing age, lifestyle, and mood. This added visual realism to the personas without using stock photos of real people.
